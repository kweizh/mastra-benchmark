Here's the complete TypeScript source code for a simple Express server, along with a summary of the review findings and recommendations for improvement.

### TypeScript Source Code

```typescript
// server.ts
import express, { Request, Response } from 'express';

const app = express();
const PORT = 3000;

// Route to handle GET requests to the root URL
app.get('/', (req: Request, res: Response) => {
    res.send('Hello, World!'); // Respond with 'Hello, World!'
});

// Basic error handling middleware
app.use((err: Error, req: Request, res: Response, next: Function) => {
    console.error(err.stack); // Log the error stack
    res.status(500).send('Something broke!'); // Respond with a 500 status
});

// Start the server
app.listen(PORT, () => {
    console.log(`Server is running on http://localhost:${PORT}`);
});
```

### Review Findings

#### Security
1. **Error Handling**: The current error handling logs the error stack, which can expose sensitive information in production.
2. **Input Validation**: No user input is currently handled, but future expansions should include validation and sanitization.

#### Performance
1. **Error Handling Middleware**: The middleware is generic; consider implementing specific error handling for different error types.
2. **Logging**: The synchronous logging can block the event loop. Asynchronous logging is recommended for production.

### Recommendations
1. **Improve Error Handling**:
   - Modify the error handling middleware to avoid exposing stack traces in production. Use an environment variable to toggle detailed error messages.
   ```typescript
   app.use((err: Error, req: Request, res: Response, next: Function) => {
       if (process.env.NODE_ENV === 'production') {
           res.status(500).send('Internal Server Error');
       } else {
           console.error(err.stack);
           res.status(500).send('Something broke!');
       }
   });
   ```

2. **Input Validation**: Use libraries like `Joi` or `express-validator` for validating incoming requests.

3. **Asynchronous Logging**: Implement a logging library like `winston` for better performance and flexibility.

4. **Environment Configuration**: Use environment variables for configuration (like the port number) instead of hardcoding them. Use the `dotenv` package for this:
   ```bash
   npm install dotenv
   ```
   Create a `.env` file:
   ```plaintext
   PORT=3000
   ```
   Load it in your server:
   ```typescript
   import dotenv from 'dotenv';
   dotenv.config();
   const PORT = process.env.PORT || 3000;
   ```

By implementing these recommendations, you can enhance the security and performance of your Express server while maintaining a clean and maintainable codebase.
